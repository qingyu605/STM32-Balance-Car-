# 平衡车项目 

## 项目概述

这是一个基于 STM32F1 系列微控制器的智能平衡车项目，采用现代控制理论实现自平衡功能。项目集成了多种传感器和通信接口，具有丰富的控制模式和扩展能力，适合作为嵌入式系统开发、传感器数据处理和控制理论学习的实践平台。

## 功能特性

- **自平衡控制**：通过 MPU6050 陀螺仪传感器获取姿态数据，使用 PD 控制算法实现车身平衡
- **速度控制**：通过编码器反馈，使用 PI 控制算法调节电机速度
- **转向控制**：基于陀螺仪数据实现精确转向
- **多种姿态算法**：支持四元数、卡尔曼滤波和互补滤波三种角度获取方法
- **多模式选择**：通过编码器可选择不同工作模式
- **串口控制**：支持通过电脑串口、蓝牙和 K210 进行远程控制
- **实时监控**：OLED 显示系统状态，串口输出陀螺仪数据和速度信息
- **电池检测**：实时监测电池电压，确保系统安全运行

## 硬件需求

- STM32F1 系列微控制器（如 STM32F103C8T6）
- MPU6050 陀螺仪传感器
- 直流电机（带编码器）× 2
- L298N 或其他电机驱动模块
- OLED 显示屏（I2C 接口）
- 蜂鸣器
- 按键
- 电池（7.4V 锂电池）
- 串口模块（用于调试和控制）
- 蓝牙模块（可选，用于无线控制）
- K210 模块（可选，用于视觉识别）

## 软件需求

- 开发环境：Keil MDK 5.30+ 或 STM32CubeIDE
- STM32 HAL 库
- STM32CubeMX（用于初始化配置）
- 串口调试助手（如 SecureCRT、Putty 等）

## 安装与编译

1. **克隆项目**：
   ```bash
   git clone <项目仓库地址>
   cd Balance_Car
   ```

2. **配置环境**：
   - 使用 STM32CubeMX 打开 `Balance_Car.ioc` 文件
   - 生成代码并导入到 Keil MDK 或 STM32CubeIDE

3. **编译项目**：
   - 在 Keil MDK 中，点击 "Build" 按钮编译项目
   - 编译成功后生成 `.hex` 文件

4. **烧录代码**：
   - 使用 ST-Link 或其他编程器将编译好的代码烧录到 STM32 开发板

## 使用方法

### 系统初始化

1. 上电后，系统会自动初始化各个模块，包括 OLED、定时器、串口、电机和陀螺仪
2. 初始化完成后，OLED 会显示 "System Init Ready!"

### 模式选择

1. 系统启动后会进入模式选择界面
2. 通过旋转编码器选择工作模式
3. 按下按键确认选择，进入所选模式

### 串口控制

1. 按下按键开启串口控制模式
2. 通过串口发送以下命令控制平衡车：
   - `LED_ON`：打开 LED
   - `LED_OFF`：关闭 LED
   - `BEEP_ON`：打开蜂鸣器
   - `BEEP_OFF`：关闭蜂鸣器
   - `speed=左轮速度,右轮速度`：控制左右轮速度（支持正负值）
   - `pid mode=0/1`：选择 PID 控制模式（0 为位置式，1 为增量式）
   - `grov check=0/1`：控制陀螺仪数据输出（0 关闭，1 开启）

### 自平衡模式

1. 在自平衡模式下，系统会自动保持车身平衡
2. 可通过串口命令调整速度和转向
3. 系统会实时监测电池电压，确保安全运行

## 核心功能实现

### 自平衡控制

- **传感器数据获取**：通过 MPU6050 陀螺仪传感器获取姿态数据
- **角度计算**：支持四元数、卡尔曼滤波和互补滤波三种角度获取方法
- **平衡控制**：使用 PD 控制算法计算平衡所需的 PWM 值
- **速度控制**：使用 PI 控制算法调节电机速度
- **转向控制**：基于陀螺仪数据实现精确转向

### 通信接口

- **USART1**：与电脑串口通信，用于调试和控制
- **USART2**：与蓝牙模块通信，用于无线控制
- **UART5**：与 K210 模块通信，用于视觉识别

### 中断处理

- **定时器中断**：用于 PID 控制和系统状态更新
- **外部中断**：用于处理 MPU6050 数据就绪中断
- **串口中断**：用于处理串口数据接收

## 注意事项

1. **安全第一**：在调试和使用过程中，确保平衡车有足够的空间，避免碰撞
2. **电池管理**：使用合适的电池，避免过充或过放
3. **参数调整**：根据实际硬件情况调整 PID 参数，以获得最佳平衡效果
4. **代码修改**：修改代码时注意保持中断处理函数的简洁性，避免影响系统实时性
